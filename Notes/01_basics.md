
## What is TypeScript?

TypeScript is a programming language developed by Microsoft.  
It is a **superset of JavaScript**, meaning it is built on top of JavaScript and includes everything that JavaScript has, plus extra powerful features.

TypeScript mainly adds **static typing**, better structure, and professional development support to JavaScript.

In short:
TypeScript = JavaScript + Types + Safety + Better Development Experience


## Why TypeScript Was Created?

JavaScript was designed mainly for small scripts.  
But today JavaScript is used for:

- Large web applications
- Backend servers
- Mobile apps
- Enterprise-level applications
- Complex projects

As applications became larger, JavaScript started facing problems like:

- Errors appear only at runtime
- No type safety
- Difficult debugging
- Complex code management
- Hard to maintain large projects
- Lack of structure in teams

To solve these issues, TypeScript was introduced.

## How TypeScript Works (Conceptually)

TypeScript does not run directly in the browser or Node.js.  
Instead, the working flow is:

1. Developer writes code in a TypeScript file.
2. TypeScript checks and validates the code.
3. TypeScript converts its code into normal JavaScript.
4. The converted JavaScript runs in browsers or Node.js.

So in the end, TypeScript still becomes JavaScript before execution.
